Technical Highlight

문제 해결 과정과 결과

Problem

배송확정 페이지에서 5만건 이상 주문 데이터 조회 시 57초 소요. 물류 운영팀의 업무 병목 발생.

Solution

서버 사이드 페이지네이션으로 API 구조 재설계. 불필요한 중복 요청 제거 및 데이터 로딩 전략 개선.

Result

조회 시간 57초 → 5초로 단축. 91% 성능 개선 달성.

Problem

3PL 물류 확장으로 4개 화주사 동시 운영 필요. 화주별 비즈니스 로직과 API가 모두 다름.

Solution

화주별 API 라우팅 분기 설계 및 공통 컴포넌트 추상화. 브랜드 독립적인 모듈 구조 구축.

Result

4개 화주사(PAF, WAAC, BOSE, BNW)를 단일 시스템에서 안정적으로 운영.
