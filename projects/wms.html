담당 역할

  • 프론트엔드 전체 아키텍처 설계 및 개발 (50개+ 관리 화면)
  • B2C 주문 처리, 3PL 입출고, 재고 관리, PDA 인터페이스 개발
  • 4개 화주사(브랜드)별 맞춤 비즈니스 로직 구현
  • Ant Design 기반 커스텀 컴포넌트 시스템 구축
  • 배송확정, 마감 등 물류 핵심 프로세스 화면 개발
  • Spring Boot API 연동 및 Oracle 쿼리 최적화 참여

주요 기능

입고 관리

발주 등록 → 입고 예정 → 검수 → 입고 확정까지의 전체 입고 프로세스. 바코드/PDA 스캔 연동을 통한 실물 검수 지원.

출고 관리

주문 수집 → 피킹 → 패킹 → 출고확정 → 배송확정까지의 아웃바운드 프로세스. 대량 주문 일괄 처리 및 송장 출력 연동.

재고 관리

로케이션별 재고 현황, 재고 조정, 재고 이동, 순환 재고 조사. 실시간 재고 동기화 및 리포트.

대시보드 & 리포트

입출고 현황, 재고 수량 집계, 마감 리포트 등 운영에 필요한 통계/리포트 화면. 기간별 필터링 및 엑셀 내보내기.

기술적 챌린지

Problem

50개 이상의 관리 화면에서 반복되는 CRUD 패턴과 테이블/폼 구조로 인한 코드 중복 및 유지보수 어려움

Solution

Ant Design 기반의 추상화된 테이블/폼 컴포넌트 시스템 구축. 공통 검색 조건, 페이지네이션, 엑셀 내보내기를 래핑한 재사용 가능한 레이아웃 패턴 설계.

Result

신규 화면 개발 시간을 절반 이하로 단축. 4개 화주사 확장 시에도 일관된 UX 유지.

Problem

대량 데이터 테이블(수천 행)에서의 렌더링 성능 저하 및 사용자 경험 악화

Solution

가상 스크롤 적용, API 페이지네이션 최적화, 검색 조건 디바운싱으로 불필요한 리렌더링 방지

Result

대량 데이터 화면에서의 초기 로딩 시간 개선 및 인터랙션 지연 해소
