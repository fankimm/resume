MH Save Editor

2017 Personal

몬스터헌터 게임의 세이브 파일을 헥스 에디터로 분석하여 바이너리 데이터 구조를 역공학. 메모리 오프셋 분석을 통해 특정 값을 수정할 수 있는 세이브 에디터를 C/C++로 구현.

C/C++Hex AnalysisReverse Engineering Binary ParsingMemory Offset

프로젝트 배경

게임 세이브 파일의 내부 구조에 대한 호기심에서 시작. 공식 문서나 SDK 없이 순수하게 바이너리 파일을 분석하여 데이터 구조를 파악하고, 특정 값을 수정하는 도구를 만들었습니다. 프로그래밍 초기에 진행한 프로젝트로, 저수준 데이터 처리와 바이너리 분석의 기초를 다지는 계기가 되었습니다.

분석 과정

1. 바이너리 분석

HxD 등의 헥스 에디터로 세이브 파일을 열어 데이터 패턴을 관찰. 게임 내 값을 변경한 후 세이브 파일의 diff를 비교하여 오프셋 위치 특정.

2. 데이터 구조 매핑

반복적인 비교 분석을 통해 캐릭터 스탯, 아이템, 장비, 재화 등의 오프셋 테이블 구축.

// 세이브 파일 오프셋 구조 (예시) Offset 0x0000 ~ 0x000F : Header / Checksum Offset 0x0010 ~ 0x001F : Character Name (UTF-8) Offset 0x0048 ~ 0x004B : Zenny (uint32, LE) Offset 0x0050 ~ 0x0053 : HR Points (uint32, LE) Offset 0x0100 ~ 0x03FF : Equipment Slots

3. 에디터 구현

C/C++로 바이너리 파일 I/O를 구현하고, 특정 오프셋의 값을 읽고 쓰는 CLI 도구 개발. 체크섬 검증 우회를 위한 재계산 로직 포함.

배운 것

  • 바이너리 파일 구조와 엔디안(Little/Big Endian)에 대한 이해
  • 메모리 오프셋 기반의 저수준 데이터 접근
  • C/C++의 파일 I/O와 포인터 조작
  • 체크섬/해시 검증 메커니즘의 이해
  • 문서 없이 바이너리를 분석하는 리버스 엔지니어링 방법론
